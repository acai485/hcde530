C4: APIs and Data Acquisition

I registered for a personal use key from TMDB and called its API. I created a .env file and used the .gitignore feature to allow my local code to use the key, while simultaneously keeping my API key from appearing on my Github. The endpoint returns data of which shows and movies are currently trending, which movies and shows are rated highest by the public, and more. I filtered for the top 10 trending movies, top 10 trending shows, and top 20 highest rated shows. I was interested in extracting these fields because I was curious to see how updated the TMDB API is, and wanted to see the API's defined trending and top rated media to compare to my own up to date knowledge.

This week I learned about the basics of how APIs work, and how they can be super useful for research purposes if I execute the extraction of data from them properly. APIs are a highly populated and updated resource, and I will definitely look at relevant ones in the future related to my UX projects. Especially with the detailed API guide and reference, it is easy to deduce how reliable and relevant the data is before choosing to analyze it.
